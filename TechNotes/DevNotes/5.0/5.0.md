2.卡顿分析
通过 profile 工具测试，发现城市指南列表的卡顿，原来是由于轮播图所用的 UICollectionView 的 item 个数达到上万个，从而到导致一些代理方法和 getter 频繁被调用，消耗了大量的 CPU
